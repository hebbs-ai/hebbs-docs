All commands connect to the HEBBS server specified by `--address` (default: `localhost:50051`).

## Global Flags

| Flag | Env Var | Default | Description |
|---|---|---|---|
| `--address` | `HEBBS_ADDRESS` | `localhost:50051` | Server address |
| `--timeout` | `HEBBS_TIMEOUT` | `10s` | Request timeout |
| `--format` | `HEBBS_FORMAT` | `human` | Output format: `human`, `json`, `raw` |
| `--tls-cert` | `HEBBS_TLS_CERT` | — | Path to TLS certificate |

## remember

Store a new memory.

```bash
hebbs-cli remember \
  --entity customer-42 \
  --content "Customer prefers email over phone" \
  --kind episodic
```

| Flag | Required | Description |
|---|---|---|
| `--entity` | Yes | Entity scope |
| `--content` | Yes | Memory content |
| `--kind` | No | `episodic`, `semantic`, or `procedural` |
| `--metadata` | No | Key=value pairs (repeatable) |

## recall

Query for relevant memories.

```bash
hebbs-cli recall "How does the customer prefer to be contacted?" \
  --strategy similarity \
  --top-k 5
```

| Flag | Required | Description |
|---|---|---|
| `cue` (positional) | Yes | Natural language query text |
| `-s`, `--strategy` | No | `similarity`, `temporal`, `causal`, `analogical` (default: `similarity`) |
| `-k`, `--top-k` | No | Max results (default: 10) |
| `-e`, `--entity-id` | No | Entity scope (required for temporal strategy) |
| `--max-depth` | No | Graph traversal depth (causal strategy). Default: `5`. |
| `--seed` | No | Seed memory ID (causal strategy). Auto-detects from cue when omitted. |
| `-w`, `--weights` | No | Scoring weights as `R:T:I:F` (see below). Default: `0.5:0.2:0.2:0.1`. |

#### Advanced Strategy Tuning

These flags let you fine-tune per-strategy behavior. Most users will not need them -- the defaults are designed for broad, accurate recall.

| Flag | Default | Description |
|---|---|---|
| `--ef-search` | `50` | HNSW candidate count for similarity recall. Higher values trade latency for accuracy. |
| `--time-range` | None (unbounded) | Temporal bounds as `START,END` (microsecond timestamps). When omitted, returns all memories newest-first. |
| `--edge-types` | None (all types) | Comma-separated edge types for causal traversal (e.g. `CausedBy,FollowedBy`). |
| `--analogical-alpha` | `0.5` | Blending weight for analogical recall: `0.0` = pure structural, `1.0` = pure embedding. |
| `--cue-context` | None | Extra context appended to the cue for embedding (not stored). Helps disambiguate short cues. |

### Composite Scoring and `--weights`

Every recall result has a **composite score** -- a weighted blend of four signals:

| Signal | Position | What it measures |
|---|---|---|
| **R**elevance | 1st | Cosine similarity between query and memory |
| **R**ecency (**T**ime) | 2nd | How recently the memory was created |
| **I**mportance | 3rd | Memory's stored importance value |
| **R**einforcement (**F**requency) | 4th | How often the memory has been recalled |

The `--weights` flag takes four colon-separated floats that control how much each signal contributes. Results are sorted by the composite score.

**Examples:**

```bash
# First, store a memory to work with
hebbs-cli remember "Met with the Acme Corp team about Q2 roadmap."

# Default composite scoring (relevance-heavy)
hebbs-cli recall "Acme meeting"

# Pure semantic relevance -- ignore recency, importance, reinforcement
hebbs-cli recall "Acme meeting" --weights 1:0:0:0

# Recency-biased -- prefer recent memories
hebbs-cli recall "Acme meeting" --weights 0.2:0.6:0.1:0.1

# Importance-biased -- surface critical memories first
hebbs-cli recall "contract renewal" --weights 0.3:0.1:0.5:0.1

# Reinforcement-biased -- find recurring patterns
hebbs-cli recall "objection handling" --weights 0.2:0.1:0.2:0.5
```

The output includes both columns:

```
 # │ Score  │ Relevance │ Memory ID                  │ Kind    │ Content
───┼────────┼───────────┼────────────────────────────┼─────────┼────────────────────
 1 │ 0.6701 │ 0.8502    │ 01KK3S0KXYE22EFF489YR...  │ episode │ Met with the Acme...
```

- **Score** is the composite (what results are sorted by)
- **Relevance** is the raw cosine similarity from the embedding model

With `--weights 1:0:0:0`, Score and Relevance will be equal since the composite is 100% relevance.

See [Composite Scoring](/concepts/recall-strategies/#composite-scoring) for the full formula and signal definitions.

### Advanced Strategy Tuning Examples

```bash
# Causal recall from a specific seed, deeper traversal, only CausedBy edges
hebbs-cli recall "What led to the pricing pushback?" \
  --strategy causal \
  --seed 01KK3S0KXYE22EFF489YRTH559 \
  --max-depth 8 \
  --edge-types CausedBy

# Temporal recall within a time range (microsecond timestamps)
hebbs-cli recall "customer interactions" \
  --strategy temporal \
  --entity-id customer-42 \
  --time-range 1709251200000000,1711929600000000

# Analogical recall favoring structural similarity
hebbs-cli recall "Have we seen similar pricing dynamics?" \
  --strategy analogical \
  --analogical-alpha 0.2

# Similarity recall with higher HNSW quality for large datasets
hebbs-cli recall "contract renewal" \
  --strategy similarity \
  --ef-search 200
```

## get

Retrieve a single memory by ID.

```bash
hebbs-cli get --id mem_abc123
```

---

## revise

Update an existing memory, creating a new revision.

```bash
hebbs-cli revise \
  --id mem_abc123 \
  --content "Customer now prefers Slack over email"
```

| Flag | Required | Description |
|---|---|---|
| `--id` | Yes | Memory ID to revise |
| `--content` | No | Updated content |
| `--metadata` | No | Updated key=value pairs |

---

## forget

Permanently erase memories.

```bash
# Forget specific memories
hebbs-cli forget --entity customer-42 --ids mem_abc123,mem_def456

# Forget all memories for an entity
hebbs-cli forget --entity customer-42 --all
```

| Flag | Required | Description |
|---|---|---|
| `--entity` | Yes | Entity scope |
| `--ids` | No | Comma-separated memory IDs |
| `--all` | No | Delete all memories for the entity |

---

## prime

Preload memories for low-latency recall.

```bash
hebbs-cli prime \
  --entity customer-42 \
  --query "account history" \
  --top-k 20
```

---

## reflect

Trigger a reflection cycle.

```bash
hebbs-cli reflect --entity customer-42
```

---

## insights

Retrieve insights generated by reflection.

```bash
hebbs-cli insights --entity customer-42 --top-k 5
```

---

## subscribe

Open a real-time subscription stream. Reads lines from stdin and prints memory pushes:

```bash
hebbs-cli subscribe --entity customer-42 --strategy similarity --threshold 0.7
```

Type text and press Enter to feed context. Memory pushes are printed as they arrive. Press `Ctrl+C` to close.

---

## status

Check server health and statistics.

```bash
hebbs-cli status
```

Output includes server version, uptime, total memory count, and health status.

---

## inspect

Show detailed information about a memory, including edges and revision history.

```bash
hebbs-cli inspect --id mem_abc123
```

---

## export

Export all memories for an entity as newline-delimited JSON.

```bash
hebbs-cli export --entity customer-42 > memories.ndjson
```

---

## metrics

Display server-side performance metrics.

```bash
hebbs-cli metrics
```

Shows operation latencies (p50, p95, p99), throughput, memory usage, and index sizes.
