title: "Types Reference"
description: "All data types in the HEBBS Python SDK."
---

All types are importable from `hebbs` or `hebbs.types`. They are implemented as Python dataclasses with full type annotation support.

## Memory

The core data type representing a stored memory.

| Field | Type | Description |
|---|---|---|
| `id` | `str` | Unique memory identifier |
| `content` | `str` | The memory text |
| `entity` | `str` | Entity scope |
| `kind` | `MemoryKind` | Memory classification |
| `edges` | `list[Edge]` | Relationships to other memories |
| `metadata` | `dict[str, str]` | User-defined key-value pairs |
| `created_at` | `datetime` | Creation timestamp |
| `updated_at` | `datetime` | Last modification timestamp |
| `revision` | `int` | Revision number (starts at 1) |
| `predecessor_id` | `str \| None` | ID of the previous revision, if any |
| `score` | `float \| None` | Relevance score (only present in recall results) |

## MemoryKind

Enum classifying the type of memory.

| Variant | Description |
|---|---|
| `EPISODIC` | Event or experience — "what happened" |
| `SEMANTIC` | Fact or knowledge — "what is true" |
| `PROCEDURAL` | Process or instruction — "how to do something" |
| `INSIGHT` | Generated by the reflection pipeline |

## Edge

A directed relationship between two memories.

| Field | Type | Description |
|---|---|---|
| `target_id` | `str` | ID of the target memory |
| `edge_type` | `EdgeType` | Relationship classification |
| `weight` | `float` | Strength of the relationship (0.0–1.0) |

## EdgeType

Enum for relationship types between memories.

| Variant | Description |
|---|---|
| `RELATED_TO` | General semantic relationship |
| `CAUSED_BY` | Causal relationship |
| `PRECEDED_BY` | Temporal ordering |
| `INSIGHT_FROM` | Insight derived from source memories |
| `REVISED_FROM` | Revision lineage |

## RecallStrategy

Enum for the four recall strategies.

| Variant | Description |
|---|---|
| `SIMILARITY` | Semantic vector similarity (HNSW index) |
| `TEMPORAL` | Time-proximity ordering (B-tree index) |
| `CAUSAL` | Follow causal edges (graph adjacency) |
| `ANALOGICAL` | Cross-domain pattern matching |

## ScoringWeights

Override composite scoring weights for `recall()` and `prime()`. When omitted, the engine uses default weights.

| Field | Type | Default | Description |
|---|---|---|---|
| `w_relevance` | `float` | `0.5` | Weight for semantic relevance (cosine similarity) |
| `w_recency` | `float` | `0.2` | Weight for temporal recency |
| `w_importance` | `float` | `0.2` | Weight for stored importance value |
| `w_reinforcement` | `float` | `0.1` | Weight for recall frequency (Hebbian reinforcement) |
| `max_age_us` | `int \| None` | `None` | Max age for recency decay (microseconds). Server default: 30 days. |
| `reinforcement_cap` | `int \| None` | `None` | Cap for reinforcement scaling. Server default: 100. |

See [Composite Scoring](/concepts/recall-strategies/#composite-scoring) for the full formula.

## StrategyDetail

Per-strategy metadata for a recall result. Each strategy populates different fields.

| Field | Type | Description |
|---|---|---|
| `strategy` | `str` | Strategy name: `"similarity"`, `"temporal"`, `"causal"`, `"analogical"` |
| `relevance` | `float` | Raw relevance score for this strategy (cosine similarity for similarity strategy) |
| `distance` | `float \| None` | HNSW distance (similarity only) |
| `timestamp` | `int \| None` | Memory timestamp (temporal only) |
| `rank` | `int \| None` | Temporal ordering rank (temporal only) |
| `depth` | `int \| None` | Graph traversal depth (causal only) |
| `embedding_similarity` | `float \| None` | Vector similarity component (analogical only) |
| `structural_similarity` | `float \| None` | Structural similarity component (analogical only) |

## RecallResult

A single memory returned from a recall query, with composite score and strategy details.

| Field | Type | Description |
|---|---|---|
| `memory` | `Memory` | The recalled memory |
| `score` | `float` | Composite score (weighted blend of relevance, recency, importance, reinforcement) |
| `strategy_details` | `list[StrategyDetail]` | Per-strategy metadata including raw relevance |

## RecallOutput

The complete response from a `recall()` call.

| Field | Type | Description |
|---|---|---|
| `results` | `list[RecallResult]` | Ranked list of recalled memories (sorted by composite score) |
| `strategy_errors` | `list[StrategyError]` | Errors from individual strategies, if any |

## PrimeOutput

Response from a `prime()` call.

| Field | Type | Description |
|---|---|---|
| `results` | `list[RecallResult]` | Blended temporal + similarity results |
| `temporal_count` | `int` | Number of results from temporal component |
| `similarity_count` | `int` | Number of results from similarity component |

## ForgetResult

Response from a `forget()` call.

| Field | Type | Description |
|---|---|---|
| `deleted_count` | `int` | Number of memories erased |
| `entity` | `str` | Entity scope that was targeted |

## ReflectResult

Response from a `reflect()` call.

| Field | Type | Description |
|---|---|---|
| `insights_created` | `int` | Number of new insights generated |
| `memories_processed` | `int` | Number of source memories analyzed |
| `latency_ms` | `float` | Server-side pipeline latency |

## SubscribePush

A push notification received on a subscription stream.

| Field | Type | Description |
|---|---|---|
| `memories` | `list[RecallResult]` | Relevant memories surfaced |
| `trigger` | `str` | The text that triggered this push |
| `timestamp` | `datetime` | When the push was generated |

## HealthStatus

Server health information.

| Field | Type | Description |
|---|---|---|
| `status` | `str` | One of `healthy`, `degraded`, `unhealthy` |
| `version` | `str` | Server version string |
| `uptime_seconds` | `float` | Server uptime |
| `memory_count` | `int` | Total memories stored across all entities |
