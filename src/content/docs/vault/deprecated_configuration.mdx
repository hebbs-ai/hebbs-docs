Vault configuration lives in `.hebbs/config.toml`. All fields have sensible defaults; you only need to add entries you want to override.

## Full Default Configuration

```toml
[chunking]
split_on = "##"              # Heading level to split on
min_section_length = 50      # Sections shorter than this (chars) merge with parent

[embedding]
model = "embeddinggemma-300m"  # ONNX model name
dimensions = 768               # Embedding dimensions
batch_size = 50                # Max sections per embed batch

[watch]
ignore_patterns = [".hebbs/", ".git/", ".obsidian/", "node_modules/", "contradictions/"]
phase1_debounce_ms = 500     # Initial debounce after file event
phase2_debounce_ms = 3000    # Settle debounce before processing
burst_threshold = 20         # Events in phase1 window to trigger burst mode
burst_debounce_ms = 10000    # Extended debounce during burst (e.g. git checkout)

[output]
insight_dir = "insights/"              # Directory for reflection output
contradiction_dir = "contradictions/"  # Directory for contradiction reports
exclude_insight_dir_from_reflect = true

[scoring]
w_relevance = 0.5       # Cosine similarity weight
w_recency = 0.2         # Temporal recency weight
w_importance = 0.2      # Stored importance weight
w_reinforcement = 0.1   # Access-count reinforcement weight

[decay]
half_life_days = 30.0          # Memory strength halves every N days without access
auto_forget_threshold = 0.01   # Memories below this decay score are candidates for removal
reinforcement_cap = 100        # Maximum access count that affects scoring

[contradiction]
enabled = true          # Enable contradiction detection during ingest
candidates_k = 10       # Max neighbors to check per memory
min_similarity = 0.5    # Minimum cosine similarity to consider a pair
min_confidence = 0.35   # Minimum LLM confidence to create contradiction edges

[llm]
provider = "anthropic"                # "anthropic", "openai", "gemini", "ollama"
model = "claude-sonnet-4-20250514"
api_key_env = "ANTHROPIC_API_KEY"     # Prefer env var over raw key
# base_url = ""                       # Override for self-hosted providers
```

## Section Reference

### `[chunking]`

Controls how markdown files are split into memory sections.

| Field | Type | Default | Description |
|---|---|---|---|
| `split_on` | string | `"##"` | Heading level to split on. `"##"` splits on H2 headings, `"###"` on H3. |
| `min_section_length` | integer | `50` | Sections shorter than this (in characters) are merged with their parent section. |

### `[embedding]`

Controls the embedding model. When `--provider openai` is used during init, embedding auto-configures to `text-embedding-3-small` with the same API key. No manual setup needed for the common case.

| Field | Type | Default | Description |
|---|---|---|---|
| `model` | string | `"embeddinggemma-300m"` | Model name. Local ONNX model or API model (e.g. `text-embedding-3-small`). |
| `dimensions` | integer | `768` | Embedding vector dimensions. Must match the model (`768` for gemma, `1536` for text-embedding-3-small). |
| `batch_size` | integer | `50` | Maximum sections embedded in a single batch call. |
| `provider` | string | none | Embedding provider: `"openai"` for API embeddings, omit or `"local"` for on-device ONNX. |
| `api_key` | string | none | API key for the embedding provider. If omitted, inherits from `[llm].api_key`. |
| `api_key_env` | string | none | Environment variable name containing the API key (e.g. `"OPENAI_API_KEY"`). If both `api_key` and `api_key_env` are set, `api_key` takes precedence. |
| `base_url` | string | none | Override the API endpoint (useful for proxies or Azure). |

### `[api]`

Controls API rate limiting for LLM and embedding requests.

| Field | Type | Default | Description |
|---|---|---|---|
| `max_concurrent_requests` | integer | `10` | Maximum concurrent API requests during indexing. Lower to 1-2 for new/low-tier API accounts. |

If you hit rate limit errors (429) during `hebbs index`, lower this value:

```toml
[api]
max_concurrent_requests = 2
```

### `[watch]`

Controls file watching behavior for `hebbs watch` and the daemon.

| Field | Type | Default | Description |
|---|---|---|---|
| `ignore_patterns` | string[] | See above | Glob patterns for files/directories to ignore. |
| `phase1_debounce_ms` | integer | `500` | Initial debounce window after a file event. |
| `phase2_debounce_ms` | integer | `3000` | Settle period after phase 1 before processing. |
| `burst_threshold` | integer | `20` | If more than this many events arrive in phase 1, extend to burst debounce. |
| `burst_debounce_ms` | integer | `10000` | Extended debounce during burst operations (e.g. `git checkout`). |

The two-phase debounce prevents unnecessary re-indexing during rapid file changes. Phase 1 collects events, phase 2 waits for the directory to settle. Burst mode handles large operations like branch switches.

### `[output]`

Controls where generated files are written.

| Field | Type | Default | Description |
|---|---|---|---|
| `insight_dir` | string | `"insights/"` | Directory for reflection output files, relative to vault root. |
| `contradiction_dir` | string | `"contradictions/"` | Directory for contradiction reports, relative to vault root. |
| `exclude_insight_dir_from_reflect` | bool | `true` | Exclude insight directory from reflect input to prevent feedback loops. |

### `[scoring]`

Controls the composite scoring weights for recall. Weights do not need to sum to 1.0; they are normalized internally.

| Field | Type | Default | Description |
|---|---|---|---|
| `w_relevance` | float | `0.5` | Weight for cosine similarity between query and memory embeddings. |
| `w_recency` | float | `0.2` | Weight for temporal recency. |
| `w_importance` | float | `0.2` | Weight for stored importance value. |
| `w_reinforcement` | float | `0.1` | Weight for access-count reinforcement. |

### `[decay]`

Controls memory decay and automatic forgetting.

| Field | Type | Default | Description |
|---|---|---|---|
| `half_life_days` | float | `30.0` | Memory strength halves every N days without access. |
| `auto_forget_threshold` | float | `0.01` | Memories with decay score below this are candidates for auto-forget. |
| `reinforcement_cap` | integer | `100` | Maximum access count that affects reinforcement scoring. |

### `[contradiction]`

Controls contradiction detection during ingest. See [Contradiction Detection](/vault/contradiction-detection/).

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | bool | `true` | Enable contradiction detection. |
| `candidates_k` | integer | `10` | Maximum nearest neighbors to check per ingested memory. |
| `min_similarity` | float | `0.5` | Minimum cosine similarity to consider two memories as potentially contradictory. |
| `min_confidence` | float | `0.35` | Minimum LLM confidence to create contradiction edges. |

### `[llm]`

LLM configuration for all autonomous features: proposition extraction, contradiction resolution, and reflection. This section is required. Vaults without LLM configuration cannot run `reflect` or autonomous contradiction detection.

For migration from older configs, `[reflect_llm]` is accepted as an alias (`#[serde(alias = "reflect_llm")]`).

| Field | Type | Default | Description |
|---|---|---|---|
| `provider` | string | — | Provider: `"anthropic"`, `"openai"`, `"gemini"`, `"ollama"`. |
| `model` | string | — | Model identifier (e.g. `"claude-sonnet-4-20250514"`, `"gpt-4o-mini"`). |
| `api_key` | string | — | API key. Prefer `api_key_env` for security. |
| `api_key_env` | string | — | Environment variable holding the API key (e.g. `"ANTHROPIC_API_KEY"`). |
| `base_url` | string | — | Base URL override for self-hosted providers. |

## Editing Configuration

Edit `.hebbs/config.toml` directly, or use the `hebbs config` command:

```bash
hebbs config show                          # Print full config
hebbs config get llm.provider              # Get a single value
hebbs config set llm.model gpt-4o-mini     # Set a single value
```

Changes take effect on the next command or watch cycle. When using the daemon, the panel can trigger a live config reload.

## Validation

HEBBS validates configuration on load. Invalid values produce clear error messages:

```
error: invalid config
  scoring.w_relevance: must be >= 0
  watch.phase1_debounce_ms: must be >= 50
```
